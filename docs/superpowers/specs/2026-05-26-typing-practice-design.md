# 한/영 타자 연습 웹프로그램 설계

## 목표

한글과 영어 타자 연습을 제공하는 웹프로그램을 구현한다. 사용자는 닉네임과 포켓몬 프로필을 선택하고, 한글 또는 영어 문장 세트를 끝까지 입력한다. 연습 완료 후 타수와 정확도를 계산해 Vercel 기반 공용 랭킹에 등록한다.

## 결정 사항

- 프레임워크: Next.js
- 배포: Vercel
- 랭킹 저장소: Vercel KV
- 랭킹 범위: 전체 사용자 공용 랭킹
- 연습 완료 기준: 문장 세트 완료형
- 프로필: 추천 포켓몬 목록에서 선택
- 포켓몬 확장 방식: 코드의 프로필 배열에 후보 객체를 추가하면 화면에 자동 반영
- 원격 저장소: `https://github.com/kms5378/Typing-practice.git`

## 사용자 흐름

1. 사용자가 닉네임을 입력한다.
2. 추천 포켓몬 프로필 중 하나를 선택한다.
3. 한글 또는 영어 연습 모드를 선택한다.
4. 문장 세트를 순서대로 입력한다.
5. 모든 문장을 완료하면 결과 화면을 보여준다.
6. 정확도 기준을 통과한 기록을 랭킹에 등록한다.
7. 언어별 Top 10 랭킹을 보여준다.

## 화면 구성

### 프로필 선택

- 닉네임 입력 필드
- 포켓몬 프로필 카드 그리드
- 선택된 프로필 강조 표시
- 시작 버튼

포켓몬 프로필은 `pokemonProfiles` 배열로 관리한다. 각 항목은 `id`, `nameKo`, `nameEn`, `imageUrl`, `accentColor`를 가진다. 첫 버전은 12종 이상을 제공하고, 후보를 늘릴 때 배열에 객체만 추가하면 된다.

### 연습 화면

- 한글/영어 모드 탭
- 현재 문장
- 입력 영역
- 진행률
- 실시간 정확도
- 경과 시간
- 현재 타수

입력값이 현재 문장과 일치하면 다음 문장으로 넘어간다. 오타는 즉시 시각적으로 표시하되, 사용자가 직접 수정할 수 있게 한다.

### 결과와 랭킹

- 최종 타수
- 정확도
- 총 입력 시간
- 완료한 문장 수
- 선택한 프로필
- 언어별 Top 10 랭킹

랭킹에는 닉네임, 포켓몬 ID, 포켓몬 이미지 URL, 언어, 타수, 정확도, 완료 시간, 등록 시각을 저장한다.

## 타수 계산

타수는 CPM 기준으로 계산한다.

```
타수 = 올바르게 입력한 문자 수 / 경과 시간(분)
```

정확도는 전체 입력 문자 대비 올바른 문자 비율로 계산한다. 랭킹 등록은 정확도 80% 이상인 기록만 허용한다. 이 기준은 낮은 품질의 무작위 입력이 랭킹에 올라가는 것을 막기 위한 최소 검증이다.

## 데이터 모델

### TypingResult

```ts
type TypingResult = {
  nickname: string;
  pokemonId: number;
  pokemonName: string;
  pokemonImageUrl: string;
  language: "ko" | "en";
  cpm: number;
  accuracy: number;
  elapsedMs: number;
  completedSentences: number;
  createdAt: string;
};
```

### KV 키

- `rankings:ko`
- `rankings:en`

각 언어별 랭킹을 분리 저장한다. API는 저장 후 해당 언어의 상위 10개 기록을 반환한다.

## API

### `GET /api/rankings?language=ko|en`

선택한 언어의 Top 10 랭킹을 반환한다.

### `POST /api/rankings`

완료 결과를 검증하고 랭킹에 저장한다.

검증 규칙:

- 닉네임은 1자 이상 20자 이하
- 언어는 `ko` 또는 `en`
- 타수는 양수
- 정확도는 80 이상 100 이하
- 포켓몬 ID는 서버가 아는 프로필 목록에 있어야 함

## 오류 처리

- KV 환경 변수가 없으면 API는 명확한 설정 오류를 반환한다.
- 랭킹 저장 실패 시 결과 화면은 유지하고, 저장 실패 메시지를 보여준다.
- 랭킹 조회 실패 시 빈 랭킹 상태와 재시도 버튼을 보여준다.
- 입력 중 새로고침하면 진행 중 기록은 복구하지 않는다.

## 배포

Vercel 프로젝트에 GitHub 저장소를 연결한다. Vercel KV를 생성한 뒤 필요한 환경 변수를 프로젝트에 연결한다. `main` 브랜치에 push하면 Vercel이 자동 배포하도록 구성한다.

필수 환경:

- Vercel KV 연결로 생성되는 KV 관련 환경 변수

## 테스트 전략

- 타수와 정확도 계산 유닛 테스트
- 랭킹 API 검증 로직 테스트
- 프로필 목록 확장 시 렌더링되는지 확인하는 컴포넌트 테스트
- Playwright로 프로필 선택부터 문장 세트 완료, 결과 표시까지 핵심 흐름 확인

## 범위 제외

- 사용자 계정 로그인
- 실시간 멀티플레이
- 포켓몬 검색 기능
- 진행 중 연습 상태 복구
- 관리자용 랭킹 삭제 UI
